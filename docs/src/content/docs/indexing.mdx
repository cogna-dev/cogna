# Bundle 与结果

这一页属于 #strong[Reference]：当你需要精确确认产物、文件或结果结构时，查这里。

## `CIQ Bundle`

构建代码仓库后，当前 bundle 通常包含：

- `manifest.json`
- `declarations.ndjson`
- `symbols.ndjson`
- `software-components.ndjson`
- `sbom.cdx.json`
- `sbom.spdx.json`
- `metadata.json`
- `checksums.txt`
- `bundle.ciq.tgz`

其中，`bundle.ciq.tgz` 是完整归档；其他文件则是构建完成后落在 `dist/` 中、方便本地直接消费的物化内容。

## `CIQ OPA Bundle`

构建 `policy-bundle` 时，当前产物会以策略 bundle 的形式供 `check` 使用。当前 `dist/` 下通常会看到：

- `manifest.json`
- `opa-bundle.tar.gz`
- `policy.rules.json`
- `metadata.json`
- `checksums.txt`
- `bundle.ciq.tgz`

其中最关键的两个文件是：

- `opa-bundle.tar.gz`：OPA 原生 bundle 载荷
- `policy.rules.json`：规则目录和帮助信息的 machine-readable catalog

当前 `manifest.json` 会写成 `ciq-opa-bundle/v1`，并把入口固定为 `codeiq/compat/deny`。

## `CIQ Diff`

当前 `diff` 重点提供：

- `summary`
- `changes`
- `componentChanges`
- 可选 `testChanges`

在 change entry 中，常见字段包括：

- `kind`
- `level`
- `semanticCategory`
- `semanticDiff`
- `beforeRecord` / `afterRecord`
- `componentDiff`

当前 repo 已经为多种生态输出 richer semantic categories，例如：

- Go：receiver、pointer receiver、method set、signature 变化
- Rust：`unsafe`、extern ABI、where clause、signature 变化
- Terraform：provider ref、lifecycle、meta-argument 变化
- OpenAPI：HTTP method、requiredness、status code、media type 变化

## `SARIF`

`check` 输出 `check.sarif.json`，当前用于把 diff + policy 结果交给审查或 CI 消费。

当前最小 SARIF 会稳定映射：

- `ruleId`
- `level`
- `message.text`
- `locations[].physicalLocation.artifactLocation.uri`
- `tool.driver.rules[].helpUri`

默认输出位置通常是 diff 文件所在目录，例如：

```text
./dist/check.sarif.json
```

## `CIQ Query` / `CIQ Result`

`query` 输入 `CIQ Query`，输出 `CIQ Result`。

一个最小 query 例如：

```json
{
  "schemaVersion": "ciq-query/v1",
  "purl": "pkg:cargo/tokio@1.43.0",
  "selector": {
    "path": "tokio::sync::Mutex::lock",
    "kind": "function"
  },
  "intent": "upgrade"
}
```

当前默认结果文件为：

```text
./query.result.json
```
