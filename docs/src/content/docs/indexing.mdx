# Bundle 与结果文件参考

这里列出 Cogna 每个命令生成的具体文件，以及每个文件中包含的内容。当你不确定某个产物是什么、在哪里、有什么字段时，查这里。

---

## 普通仓库 Bundle（CIQ Bundle）

在目标仓库目录执行 `cogna build` 后，`dist/` 下会生成：

```text
dist/
├── manifest.json           ← bundle 元数据：purl、profile、生成时间
├── declarations.ndjson     ← 公开声明列表（函数、类型、接口等）
├── symbols.ndjson          ← 符号信息（路径、签名、位置、文档）
├── software-components.ndjson  ← 软件组件依赖列表
├── sbom.spdx.json          ← SPDX 格式的 SBOM
├── metadata.json           ← 补充元数据
├── checksums.txt           ← 文件校验和
└── bundle.ciq.tgz          ← 完整归档（cache / 分发用）
```

其中 `bundle.ciq.tgz` 是 cache 复用与分发的主要文件；如果这是 policy 仓库，它也会作为 `checks.policy` 指向的 policy bundle。

对 Go / Rust 仓库，`metadata.json` 中记录的是 canonical author path 的结果；当配置 `validation: lsp` 时，LSP 只做 parity validation，不会再写出第二份 backend-specific bundle 或第二套 snapshot truth。

## Policy Bundle（CIQ OPA Bundle）

在 policy 仓库目录执行 `cogna build`（`profile: policy-bundle`）后，`dist/` 下会生成：

```text
dist/
├── manifest.json           ← bundle 元数据（manifest type: ciq-opa-bundle/v1）
├── opa-bundle.tar.gz       ← OPA 原生 bundle 载荷（给 opa eval 用）
├── policy.rules.json       ← 规则目录：ruleId、summary、level、helpUri
├── metadata.json
├── checksums.txt
└── bundle.ciq.tgz          ← 给 `checks.policy` 引用的文件
```

**注意：** 当前 `cogna check` 不再从 CLI 读取 `--policy` 参数，而是从 `cogna.yml -> checks.policy` 读取 policy bundle。真正要引用的文件仍然是 `bundle.ciq.tgz`，不是 `opa-bundle.tar.gz`。

## Diff 文件（CIQ Diff）

执行 `cogna diff` 后，`./sdk/dist/diff.json` 的结构：

```json
{
  "schemaVersion": "ciq-diff/v1",
  "summary": {
    "added": 2,
    "changed": 1,
    "removed": 0
  },
  "changes": [
    {
      "kind": "changed",
      "level": "error",
      "path": "Client.Do",
      "semanticDiff": {
        "findings": [
          {
            "ruleId": "compat.go.signature-changed",
            "level": "error",
            "message": "parameter 'ctx context.Context' was added"
          }
        ]
      },
      "beforeRecord": { "uri": "client/client.go", "startLine": 87 },
      "afterRecord": { "uri": "client/client.go", "startLine": 89 }
    }
  ],
  "componentChanges": [
    {
      "kind": "changed",
      "beforePurl": "pkg:cargo/serde@1.0.195",
      "afterPurl": "pkg:cargo/serde@1.0.197",
      "beforeVersion": "1.0.195",
      "afterVersion": "1.0.197"
    }
  ]
}
```

## SARIF 文件（check.sarif.json）

执行 `cogna check` 后，`./sdk/dist/check.sarif.json` 符合 SARIF 2.1.0 标准：

```json
{
  "version": "2.1.0",
  "runs": [{
    "tool": {
      "driver": {
        "name": "Cogna",
        "rules": [
          {
            "id": "compat.go.signature-changed",
            "helpUri": "https://cogna.xaclabs.dev/docs/policies/generated/go/signature-changed"
          }
        ]
      }
    },
    "results": [
      {
        "ruleId": "compat.go.signature-changed",
        "level": "error",
        "message": { "text": "function 'Client.Do' added parameter 'ctx'" },
        "locations": [{
          "physicalLocation": {
            "artifactLocation": { "uri": "client/client.go" },
            "region": { "startLine": 87 }
          }
        }]
      }
    ]
  }]
}
```

## Query 文件（CIQ Query / CIQ Result）

**查询文件格式（输入）：**

```json
{
  "schemaVersion": "ciq-query/v1",
  "purl": "pkg:cargo/tokio@1.43.0",
  "selector": {
    "path": "tokio::sync::Mutex::lock",
    "kind": "function"
  },
  "intent": "upgrade"
}
```

执行 `cogna query ./query.json` 后，结果保存在 `./query.result.json`。
