# 理解 CodeIQ 产物：Config / Bundle / OPA Bundle / Diff / Result / SARIF

很多用户第一次接触 CodeIQ 时，真正困惑的不是命令本身，而是：为什么会有这么多产物？

理解它们最好的方式不是背字段，而是问三个问题：

1. #strong[谁生成它？]
2. #strong[谁消费它？]
3. #strong[我在什么场景下需要关心它？]

## 总览：一张表先看全

| 产物 | 谁生成 | 谁消费 | 什么时候你需要关心它 |
| --- | --- | --- | --- |
| `CIQ Config` | `init` | `build` 等 CLI 流程 | 第一次接入仓库时 |
| `CIQ Bundle` | `build` | `query`、MCP、publish、diff | 你想查询或复用仓库事实时 |
| `CIQ OPA Bundle` | `build`（policy-bundle） | `check` | 你想做 policy check 时 |
| `CIQ Diff` | `diff` | `check`、人工 review | 你想知道这次改了什么时 |
| `CIQ Result` | `query` / MCP | 人、脚本、AI | 你想查具体接口时 |
| `SARIF` | `check` | CI、审查流程、人 | 你想把检查结果接进流程时 |

## `CIQ Config`：告诉 CodeIQ 这个仓库是什么

这是工作流的起点。

你可以把它理解为：

- 当前仓库属于哪个 profile；
- 当前仓库的 PURL 是什么；
- 从哪些文件构建；
- 默认检查策略应该怎么接。

对普通用户来说，第一次接入时关心它就够了。后续大多数时候，你只是在消费它带来的行为，而不是频繁手改所有字段。

## `CIQ Bundle`：一份可查询、可比较、可复用的仓库快照

这是 CodeIQ 最核心的产物。

一个 bundle 不只是“索引文件”，而是一份把仓库公开事实整理后的结构化快照。当前代码 bundle 通常包含：

- `declarations.ndjson`
- `symbols.ndjson`
- `software-components.ndjson`
- `sbom.cdx.json`
- `sbom.spdx.json`
- `metadata.json`
- `checksums.txt`

你应该在这些场景关心 bundle：

- 你想查询某个接口；
- 你想把同一份事实复用给 CLI / MCP / 自动化；
- 你想把当前版本和另一个版本做结构化比较。

## `CIQ OPA Bundle`：给 `check` 用的 policy 载荷

这不是给 AI 查询的 bundle，而是给发布前检查使用的本地 policy bundle。

在当前实现里，它的活跃路径应该理解为：

- 本地准备 policy bundle；
- `check` 调用本机 `opa eval --bundle ...`；
- 再把结果转成当前的本地 `SARIF` 输出。

因此，它的价值不是“存放更多代码事实”，而是“给变更事实附加组织自己的规则”。

## `CIQ Diff`：不是源码 patch，而是结构化变更结果

源码 diff 告诉你“改了哪些行”；`CIQ Diff` 告诉你“改了哪些公开接口、契约和软件成分”。

在当前基线里，用户更值得关心的是：

- `kind`
- `level`
- `semanticCategory`
- `semanticDiff`
- `beforeRecord` / `afterRecord`
- `componentDiff`

如果你想做发布前判断，这会比单纯看源码 patch 更接近真正的问题。

## `CIQ Result`：把查询结果结构化返回

无论你是通过一次性 `query`，还是通过 local query runtime 去查，最后都需要一个稳定的结果载体。

`CIQ Result` 的意义在于：

- 给脚本消费时足够结构化；
- 给人看时足够可追溯；
- 给 AI 用时不会丢掉 path、signature、location 等关键信息。

## `SARIF`：把检查结果带进审查流程

`SARIF` 是 `check` 的最终输出层。

对用户来说，你可以简单把它理解成：CodeIQ 把 diff + policy 的结果整理成一份更容易进入现有工作流的报告格式。

当前口径应理解为：#strong[local minimal SARIF v2.1.0 output]。

这句话的重点在于：

- 它已经足够支持当前本地工作流；
- 但它不意味着“任何 hosted 平台都零调整自动兼容”。

## 一条更容易理解的心智模型

如果你只想记一句话，可以这样记：

```text
Config 告诉 CodeIQ 怎么开始，Bundle 记录当前事实，Diff 记录变化，Check + SARIF 记录结论，Result 负责查询返回。
```
