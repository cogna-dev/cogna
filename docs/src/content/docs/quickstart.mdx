# 快速开始

这一页属于 #strong[Tutorials]：跟着步骤做，先跑通一遍，再读其他文档。

## 你会完成什么？

完成这页后，你会拿到：

- 一个 `codeiq.yaml`
- 一个构建好的 bundle
- 一个 diff 文件
- 一个 SARIF 检查结果
- 一次真实查询结果

## 前置条件

- 已安装 `codeiq`
- 有一个本地仓库可供测试
- 如果要执行 `check`，本机还需要 `opa`
- 如果你还没有 policy bundle，准备一个单独的 policy repo，或使用已经构建好的本地 policy bundle

## 第 1 步：初始化

```bash
codeiq init
```

这一步会生成 `codeiq.yaml`。

第一次使用时，先确认这三个字段：

- `profile`：例如 `go-module`、`rust-crate`、`terraform-module`、`openapi-spec`
- `purl`：当前仓库的唯一标识
- `checks.policy`：当前只是字符串配置；真正执行 `check` 时，仍建议显式传 `--policy <bundle-path>`

## 第 2 步：构建 bundle

```bash
codeiq build ./sdk
```

构建完成后，`dist/` 下通常会看到：

- `manifest.json`
- `declarations.ndjson`
- `symbols.ndjson`
- `software-components.ndjson`
- `sbom.cdx.json`
- `sbom.spdx.json`
- `metadata.json`
- `checksums.txt`
- `bundle.ciq.tgz`

这里的 `bundle.ciq.tgz` 是整包产物；其余文件则是已物化的 bundle 内容。

## 第 3 步：生成 diff

```bash
codeiq diff ./sdk --since <last-release-commit> --test-changes true
```

你会得到 `dist/diff.json`。当前 `CIQ Diff` 除了 `summary` 之外，还会在很多 change 上输出：

- `semanticCategory`
- `semanticDiff`
- `beforeRecord` / `afterRecord`
- `componentDiff`

## 第 4 步：准备 policy bundle

`check` 需要一个本地 policy bundle。最常见的方式有两种：

### 方式 A：使用已有 policy bundle

如果团队已经有一个构建好的 policy bundle，直接记住它的路径，例如：

```text
./policy/dist/bundle.ciq.tgz
```

### 方式 B：自己构建一个 policy repo

为单独的 policy 仓库准备 `codeiq.yaml`，把 `profile` 设为 `policy-bundle`，然后执行：

```bash
codeiq build ./policy
```

当前 `build` 会在这个 policy repo 的 `dist/` 目录下生成：

- `manifest.json`
- `opa-bundle.tar.gz`
- `policy.rules.json`
- `checksums.txt`
- `bundle.ciq.tgz`

在当前 repo reality 下，传给 `check --policy` 的通常是这个 policy repo 的 `dist/bundle.ciq.tgz`。

## 第 5 步：执行检查

```bash
codeiq check ./sdk/dist/diff.json --policy ./policy/dist/bundle.ciq.tgz
```

你会得到 `check.sarif.json`。当前实现会把它写到 diff 文件所在目录，例如这里通常是：

```text
./sdk/dist/check.sarif.json
```

## 第 6 步：执行查询

```json
{
  "schemaVersion": "ciq-query/v1",
  "purl": "pkg:cargo/tokio@1.43.0",
  "selector": {
    "path": "tokio::sync::Mutex::lock",
    "kind": "function"
  },
  "intent": "upgrade"
}
```

保存为 `query.json` 后运行：

```bash
cd ./sdk
codeiq query ../query.json
```

当前默认输出文件是：

```text
./query.result.json
```

## 你现在应该看到什么？

- `codeiq.yaml`
- `dist/manifest.json`
- `dist/diff.json`
- `dist/check.sarif.json`
- `dist/bundle.ciq.tgz`
- `query.result.json`

如果你同时构建了一个 policy repo，你还会看到：

- `policy/dist/opa-bundle.tar.gz`
- `policy/dist/policy.rules.json`
- `policy/dist/bundle.ciq.tgz`
