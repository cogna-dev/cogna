# 快速开始

这份教程会带你从零跑完一次完整的本地 Cogna 工作流。完成后你手里会有：一份配置文件、一个 bundle 快照、一份 diff 报告、一份 SARIF 检查结果。

## 前置条件

- 已安装 `cogna` CLI
- 有一个本地仓库（Go / Rust / Terraform / OpenAPI 都可以）
- 如果要运行 `check`，需要额外安装 [`opa`](https://www.openpolicyagent.org/docs/latest/#running-opa)

---

## 第 1 步：初始化配置

在你的仓库根目录运行：

```bash
cogna init
```

这会生成一个 `cogna.yml`。打开它，先确认这几个字段：

```yaml
schemaVersion: ciq-config/v2
profile: go-module          # 改成你仓库对应的类型
purl: pkg:golang/github.com/your-org/your-lib@1.0.0   # 改成你的 PURL
source:
  repo: https://github.com/your-org/your-lib
  ref: main
inputs:
  include:
    - "**/*.go"
cache:
  type: local
  local:
    storeDir: .cogna/cache
```

`profile` 决定 Cogna 如何理解你的代码。支持的值：`go-module`、`rust-crate`、`terraform-module`、`openapi-spec`、`policy-bundle`。

---

## 第 2 步：构建 bundle

```bash
cd ./your-lib
cogna build
```

构建完成后，`dist/` 目录会有这些文件：

```text
dist/
├── manifest.json           ← bundle 的元数据描述
├── declarations.ndjson     ← 公开声明（函数、类型、接口等）
├── metadata.json
├── checksums.txt
└── bundle.ciq.tgz          ← 完整归档，后续步骤用的就是它
```

`bundle.ciq.tgz` 是你所有后续操作的基础——diff、check，以及后续 cache 复用都会用到它。

---

## 第 3 步：生成 diff

假设你已经准备好了基线版本和当前版本两套产物，想看它们之间改了什么：

```bash
cd ./your-lib
# 先把旧版本产物放到 dist/base，把当前产物放到 dist/target
cogna diff
```

这会生成 `dist/diff.json`。打开它，你会看到类似这样的结构：

```json
{
  "summary": {
    "total": 3,
    "added": 1,
    "removed": 0,
    "changed": 2
  },
  "changes": [
    {
      "id": "github.com/your-org/your-lib.Client.Do",
      "kind": "changed",
      "level": "warning",
      "path": "pkg/client/client.go",
      "semanticDiff": {
        "findings": [
          { "code": "go-signature-changed" }
        ]
      }
    }
  ]
}
```

`semanticDiff.findings` 是最重要的字段——它告诉你这次变化在语义上属于哪种类型（比如函数签名变了、方法集收缩了、参数变成必填了）。

---

## 第 4 步：准备 policy bundle（可选，用于 check）

`check` 命令需要一个 policy bundle 来判断哪些变化是可接受的。最简单的方式是用内置规则：

**新建一个 policy 仓库**，配置 `cogna.yml`：

```yaml
schemaVersion: ciq-config/v2
profile: policy-bundle
purl: pkg:generic/your-org/compat-policy@1.0.0
source:
  repo: https://github.com/your-org/compat-policy
  ref: main
inputs:
  include:
    - "**/*.rego"
cache:
  type: local
  local:
    storeDir: .cogna/cache
```

然后构建它：

```bash
cd ./compat-policy
cogna build
```

你会得到 `./compat-policy/dist/bundle.ciq.tgz`，这就是传给 `check` 的 policy bundle。

---

## 第 5 步：运行检查

```bash
cd ./your-lib
cogna check
```

这会生成 `./your-lib/dist/check.sarif.json`。SARIF 是行业标准格式，可以被 GitHub Code Scanning、Azure DevOps、各种代码审查平台直接消费。

打开 `check.sarif.json`，最值得关注的是：

```json
{
  "runs": [{
    "results": [
      {
        "ruleId": "compat.go.signature-changed",
        "level": "warning",
        "message": { "text": "Go public signature changed." },
        "locations": [{
          "physicalLocation": {
            "artifactLocation": { "uri": "pkg/client/client.go" },
            "region": { "startLine": 42 }
          }
        }]
      }
    ]
  }]
}
```

`level` 有三档：`error`（可能破坏调用方）、`warning`（需要评估）、`note`（纯信息）。

---

## 你现在有了什么？

```text
your-lib/dist/
├── bundle.ciq.tgz      ← 当前版本的公开接口快照
├── diff.json           ← 与上个版本的结构化变更
└── check.sarif.json    ← policy 检查结果（SARIF）
```
