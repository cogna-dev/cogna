# 分析变更

`cogna diff` 用来比较两个版本的公开接口，回答一个关键问题：

“这次改动会不会破坏使用者？”

## 准备输入

你需要两套产物：

- `dist/base`：基线版本
- `dist/target`：目标版本

然后执行：

```bash
cogna diff
```

## 输出文件

- `dist/diff.json`

重点看三类信息：

- 新增（added）
- 删除（removed）
- 变更（changed）

## 如何快速判断风险

优先看这些类型：

- 函数/方法签名变化
- 参数从可选变必填
- 返回结构收窄
- 已公开符号被删除

这些通常意味着升级风险更高。

## 与 check 的关系

- `diff` 告诉你“变了什么”
- `check` 告诉你“这些变化是否违反规则”

所以推荐顺序：

1. `build`
2. `diff`
3. `check`
