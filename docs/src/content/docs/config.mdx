# 配置与 PURL

Cogna 默认读取仓库根目录的 `cogna.yml`。如果只有旧文件名 `cogna.yaml`，当前版本仍会兼容读取，但新的初始化、文档和示例都统一使用 `cogna.yml`。

## 一个完整的 v2 配置示例

```yaml
schemaVersion: ciq-config/v2
profile: openapi-spec
purl: pkg:openapi/acme/payment-api@2026.03.01
source:
  repo: https://github.com/acme/payment-api
  ref: main
inputs:
  include:
    - openapi/**/*.yaml
checks:
  format: sarif
  policy: ./dist/policy.ciq.tgz
diff:
  includeTestChanges: false
sbom:
  format: spdx
  dependencyBundles: true
  requireLocalPackages: true
publish:
  includeDependencies: true
registry:
  port: 8787
  storeDir: .cogna/cache/registry
mcp:
  port: 3000
```

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `schemaVersion` | 是 | 当前写 `ciq-config/v2` |
| `profile` | 是 | 仓库类型：Go / Rust / Terraform / OpenAPI / policy |
| `purl` | 是 | 当前仓库或规格的唯一标识 |
| `source.repo` | 是 | 源仓库地址 |
| `source.ref` | 是 | 默认分支或基线引用 |
| `inputs.include` | 是 | 要扫描的文件 glob |
| `checks.format` | 否 | 目前固定为 `sarif` |
| `checks.policy` | 否 | `cogna check` 默认读取的 policy bundle 路径 |
| `diff.includeTestChanges` | 否 | `diff` 是否包含测试变化 |
| `sbom.format` | 否 | 当前仅支持 `spdx` |
| `sbom.dependencyBundles` | 否 | 是否为可解析的本地依赖额外产出 dependency bundle |
| `sbom.requireLocalPackages` | 否 | 本地依赖无法解析时是否直接失败 |
| `publish.includeDependencies` | 否 | `cogna publish` 是否同时发布 dependency bundles |
| `registry.port` | 否 | `cogna registry serve` 默认端口 |
| `registry.storeDir` | 否 | 本地 registry 存储目录 |
| `mcp.port` | 否 | `cogna mcp serve` 默认端口 |

## 配置优先级

### Registry store 目录

本地 registry 存储目录按下面顺序决定：

1. 环境变量 `COGNA_REGISTRY_STORE_DIR`
2. `cogna.yml` 中的 `registry.storeDir`
3. 默认值 `.cogna/cache/registry`

这意味着你可以在容器或 CI 里直接覆盖存储目录，而不必改仓库配置：

```bash
COGNA_REGISTRY_STORE_DIR=/data/registry cogna registry serve --port 8787
```

## 支持的 profile

| profile | 用途 |
|---------|------|
| `go-module` | Go module |
| `rust-crate` | Rust crate |
| `terraform-module` | Terraform module |
| `openapi-spec` | OpenAPI 规格仓库 |
| `policy-bundle` | OPA policy bundle |

## 什么是 PURL？

PURL（Package URL）是 Cogna 用来标识 bundle、diff 基线和 registry 记录的标准格式：

```text
pkg:<type>/<namespace>/<name>@<version>
```

示例：

```text
pkg:golang/github.com/gin-gonic/gin@1.10.0
pkg:cargo/tokio@1.43.0
pkg:terraform/hashicorp/terraform-aws-vpc@5.18.1
pkg:openapi/acme/payment-api@2026.03.01
pkg:generic/acme/compat-policy@2026.03.27
```
