# 远程索引

远程索引的核心目标是：把一次构建结果复用给更多人和更多流水线。

## 快速搭建

在一台共享机器上：

```bash
cogna serve --port 8787
```

在业务仓库里配置：

```yaml
cache:
  type: http
  storeDir: .cogna/cache
  http:
    endpoint: http://cache.internal:8787
```

之后照常执行：

```bash
cogna build
```

## 你会得到什么

- 本地 miss 时自动读取远端缓存
- 命中后回填本地缓存
- CI 和本地共享同一份索引事实

## 适合谁

- 多人协作项目
- 分支较多、重复构建频繁的团队
- 希望降低 CI 时间与成本的仓库

## 与下载页的关系

如果你希望非终端用户也参与使用，可在 [/download](/download) 安装 Desktop 版本，由桌面端消费同样的索引结果。
