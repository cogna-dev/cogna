# 概览

这一页属于 #strong[Explanation]：它回答的是“CodeIQ 是什么、适合解决什么问题”，而不是“命令怎么写”。

## CodeIQ 解决什么问题？

CodeIQ 不是新的 linter，也不是 IDE / LSP 的替代品。

它关注的是：#strong[公开接口、契约和版本差异]。

当你要回答下面这些问题时，CodeIQ 会比单纯翻 README、看 diff 文本或做实现级静态分析更直接：

- 这个库或 API 当前对外承诺了什么？
- 两个版本之间哪些公开声明真的变了？
- 这些变化是否可能破坏兼容性？
- 我能否把检查结果接到本地 CI / 审查流程里？

当你面对下面这些场景时，CodeIQ 会特别有用：

- SDK / Library 发版前要确认有没有破坏兼容性
- Terraform Module 或 OpenAPI 规格更新后要看清楚变更
- 想让 AI / Agent 查询当前版本的真实接口，而不是只搜 README
- 想把变更检查结果输出成 SARIF，接到现有审查流程里

## 当前提供哪些核心能力？

| 能力 | 当前结果 |
| --- | --- |
| 构建仓库快照 | `build` 生成 `CIQ Bundle`，其中包含声明、符号、software components、SBOM、metadata、checksums 和 `bundle.ciq.tgz` |
| 构建策略快照 | `profile: policy-bundle` 时，`build` 生成 `CIQ OPA Bundle`，其中包含 `opa-bundle.tar.gz`、`policy.rules.json` 和 `bundle.ciq.tgz` |
| 比较版本 | `diff` 生成 `CIQ Diff`，当前会输出 `summary`、`changes`、`componentChanges`，并为很多 change 补上 `semanticCategory`、`semanticDiff`、`componentDiff` |
| 发布前检查 | `check` 在本机执行 `opa eval --bundle ... data.codeiq.compat.deny`，并生成本地最小 `SARIF v2.1.0` |
| 查询上下文 | `query` 输出 `CIQ Result`；`mcp` 提供本地查询 runtime，支持 AI / Agent 重复查询 |
| 共享产物 | `publish`、`registry download-local`、`registry download`、`registry serve` 让 bundle 在单机工作流里复用 |

## 当前有哪些核心对象？

- #strong[`CIQ Config`]：定义 profile、PURL、输入路径和 checks policy；
- #strong[`CIQ Bundle`]：面向代码 / 规格仓库的声明与 component 快照；
- #strong[`CIQ OPA Bundle`]：面向 policy 的策略快照，供 `check` 装载；
- #strong[`CIQ Diff`]：描述版本之间 declaration 与 component 的变化；
- #strong[`SARIF`]：`check` 的对外结果格式；
- #strong[`Local Registry`]：单机上的 bundle publish / resolve / download façade；
- #strong[`MCP runtime`]：本地查询工具层，不替代 `build / diff / check / publish`。

## 当前支持哪些仓库？

- Go module
- Rust crate
- Terraform Module
- OpenAPI specification
- OPA policy bundle

这些 profile 统一使用 PURL 作为 canonical identifier，并围绕一套本地工作流组织：配置、构建、比较、检查、查询和本地共享。

## 当前工作流

```text
init -> build -> diff -> check -> query -> publish
```

## CodeIQ 不做什么？

- 不做 hosted registry
- 不做 distributed replication 或 multi-node HA
- 不把 trust verifier / KMS 作为当前 active scope
- 不把 MCP 变成 build / diff / check / publish 的替代入口

## 当前边界

CodeIQ 当前是 #strong[local-only] 工作流：

- CLI 在本机运行
- bundle 在本机共享和下载
- query / MCP 在本机消费这些 bundle

这意味着：当前文档、命令和示例都优先描述 repo 已实现的本地路径，而不是更宽的 hosted / remote 叙事。
