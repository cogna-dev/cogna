# MCP / Cache 参考

这里记录的是**当前 reality**：MCP 仍提供 `serve` / `status`；cache 侧公开暴露的是 `serve` 与 `cache list/add/remove`，HTTP 面是最小的 cache object proxy。

---

## MCP 命令

| 命令 | 说明 |
|------|------|
| `cogna mcp serve [--port <port>]` | 在前台运行 MCP server |
| `cogna mcp status` | 查看 MCP descriptor 和健康状态 |

示例：

```bash
cogna build
cogna mcp serve --port 3000
```

---

## Cache 命令

| 命令 | 说明 |
|------|------|
| `cogna serve [--port <port>]` | 启动本地 HTTP cache proxy |
| `cogna cache list [--prefix <prefix>]` | 列出当前 workspace local cache 中的对象 |
| `cogna cache add --key <key> --file <path>` | 把本地文件加入 current local cache |
| `cogna cache remove --key <key>` | 从 current local cache 删除对象 |

### Local cache 根目录

1. `cogna.yml -> cache.local.storeDir`
2. `.cogna/cache`

示例：

```bash
cogna serve --port 8787
```

---

## HTTP Cache 接口

运行 `cogna serve` 后，可用的 API：

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/health` | 健康检查 |
| `GET` | `/cache/v1/object?key=<key>` | 读取 cache object |
| `PUT` | `/cache/v1/object?key=<key>` | 写入 cache object |

---

## Docker 运行 Cache Proxy

cache proxy 可以直接以容器方式交付：

```bash
docker build -f integrations/deployments/docker/Dockerfile -t cogna-cache-proxy:local .

docker run --rm \
  -p 8787:8787 \
  -v "$PWD/.cogna-docker-cache:/data/cache" \
  cogna-cache-proxy:local
```

容器里的 `/data/cache` 会作为 local cache 根目录，通常配合 volume 持久化。
