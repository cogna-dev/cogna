# MCP / Registry 参考

这里记录的是**当前 reality**：只有 `serve` / `status` 公开暴露给 CLI；上传、解析、下载 bundle 的行为通过 `publish` 和 registry HTTP API 完成。

---

## MCP 命令

| 命令 | 说明 |
|------|------|
| `codeiq mcp serve [--port <port>]` | 在前台运行 MCP server |
| `codeiq mcp status` | 查看 MCP descriptor 和健康状态 |

示例：

```bash
codeiq build
codeiq mcp serve --port 3000
```

---

## Registry 命令

| 命令 | 说明 |
|------|------|
| `codeiq publish` | 发布当前仓库 `dist/bundle.ciq.tgz` |
| `codeiq registry serve [--port <port>]` | 启动本地 HTTP registry server |
| `codeiq registry status` | 查看 registry descriptor 和健康状态 |

### Registry store 目录优先级

1. `CODEIQ_REGISTRY_STORE_DIR`
2. `codeiq.yml -> registry.storeDir`
3. `.codeiq/cache/registry`

示例：

```bash
CODEIQ_REGISTRY_STORE_DIR=/data/registry codeiq registry serve --port 8787
```

---

## HTTP Registry 接口

运行 `codeiq registry serve` 后，可用的 API：

| 方法 | 路径 | 说明 |
|------|------|------|
| `GET` | `/health` | 健康检查 |
| `POST` | `/api/v1/bundles` | 上传 bundle |
| `GET` | `/api/v1/bundles/resolve?purl=<purl>` | 按 PURL 查找 bundle |
| `GET` | `/api/v1/packages/{purl}/versions` | 列出某个包的已发布版本 |
| `GET` | `/api/v1/bundles/{bundleId}` | 获取 bundle 元数据 |
| `GET` | `/api/v1/bundles/{bundleId}/download` | 下载 bundle 文件 |

## Docker 运行 Registry

P6 完成后，registry server 可以直接以容器方式交付：

```bash
docker build -f integrations/deployments/docker/Dockerfile -t codeiq-registry:local .

docker run --rm \
  -p 8787:8787 \
  -e CODEIQ_REGISTRY_STORE_DIR=/data/registry \
  -v "$PWD/.codeiq-docker-registry:/data/registry" \
  codeiq-registry:local
```

容器里 `CODEIQ_REGISTRY_STORE_DIR` 必须指向可写目录，通常配合 volume 持久化。
