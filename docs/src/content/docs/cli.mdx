# CLI 参考

当前版本的 Cogna CLI 以 `cogna.yml` 为中心：`build`、`diff`、`check`、`serve`、`cache ...`、`mcp serve` 都优先读取当前工作目录对应仓库的配置，而不是要求你在命令行里重复传路径。

---

## `cogna init`

生成一个新的 `cogna.yml` 模板：

```bash
cogna init
```

默认输出到当前目录的 `./cogna.yml`。

---

## `cogna build`

读取当前仓库的 `cogna.yml`，生成 `dist/` 下的 bundle 与索引产物：

```bash
cogna build
```

典型输出：

```text
dist/
├── manifest.json
├── declarations.ndjson
├── symbols.ndjson
├── software-components.ndjson
├── sbom.spdx.json
├── metadata.json
├── checksums.txt
├── bundle.ciq.tgz
└── dependencies/              ← 仅在可解析本地依赖时出现
```

如果 `profile: policy-bundle`，还会额外生成：

```text
dist/
├── opa-bundle.tar.gz
└── policy.rules.json
```

---

## `cogna diff`

根据当前仓库配置和 `dist/base` / `dist/target` 产物生成结构化 diff：

```bash
cogna diff
```

输出文件：

```text
dist/diff.json
```

---

## `cogna check`

读取 `cogna.yml` 里的 `checks.policy` 并对 `dist/diff.json` 执行 policy 检查：

```bash
cogna check
```

输出文件：

```text
dist/check.sarif.json
```

> `check` 依赖本机可用的 `opa eval`。

---

## `cogna query`

一次性查询某个 bundle 中的符号或 operation：

```bash
cogna query <query-file>
```

示例查询文件：

```json
{
  "schemaVersion": "ciq-query/v1",
  "purl": "pkg:cargo/tokio@1.43.0",
  "selector": {
    "path": "tokio::sync::Mutex::lock"
  },
  "intent": "upgrade"
}
```

`selector.kind` 当前是可选的；如果省略，Cogna 会在服务端做 ranked matching。

---

## `cogna serve`

启动本地 HTTP cache proxy：

```bash
cogna serve --port 8787
```

当前实现暴露两个 cache object 端点：

- `GET /health`
- `GET /cache/v1/object?key=...`
- `PUT /cache/v1/object?key=...`

---

## `cogna cache`

### `cache list`

列出当前 workspace local cache 中的对象：

```bash
cogna cache list
```

也可以按前缀过滤：

```bash
cogna cache list --prefix blob/
```


### `cache add`

把一个本地文件加入当前 workspace 的 local cache：

```bash
cogna cache add --key blob/sha256/example --file ./dist/bundle.ciq.tgz
```

### `cache remove`

从当前 workspace 的 local cache 删除对象：

```bash
cogna cache remove --key blob/sha256/example
```

---

## `cogna mcp`

### `mcp serve`

在前台运行 MCP server：

```bash
cogna mcp serve --port 3000
```

### `mcp status`

查看 MCP server descriptor 和健康状态：

```bash
cogna mcp status
```

---

## 当前推荐工作流

```bash
cogna init
cogna build
cogna diff
cogna check
cogna serve --port 8787
cogna cache list
```

另一个终端里，如果要给 AI / Agent 使用：

```bash
cogna mcp serve --port 3000
```
