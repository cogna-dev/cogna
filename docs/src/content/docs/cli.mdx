# CLI

这一页属于 #strong[Reference]：当你需要查命令时，直接看这里。

## 命令总览

| 命令 | 输入 | 当前输出 |
| --- | --- | --- |
| `codeiq init` | 无 | 生成 `codeiq.yaml` |
| `codeiq build <repo>` | 代码 repo 或 policy repo | 物化 `dist/*`，并生成 `dist/bundle.ciq.tgz` |
| `codeiq diff <repo> [--since <ref>] [--test-changes <bool>]` | 已配置 repo | 生成 `dist/diff.json` |
| `codeiq check <diff-file> --policy <policy-bundle>` | `CIQ Diff` + 本地 policy bundle | 生成 `check.sarif.json` |
| `codeiq query <query-file>` | `CIQ Query` | 生成 `query.result.json` |
| `codeiq publish <bundle-file>` | `bundle.ciq.tgz` | 生成 publish receipt |
| `codeiq registry ...` | PURL / bundle / upload request | 本地 registry metadata 或物化后的 bundle |
| `codeiq mcp ...` | 本地 bundle context | 本地 MCP runtime |

## 子命令参考

### `registry`

- `codeiq registry upload-local <upload-request-file>`
- `codeiq registry download-local <purl>`
- `codeiq registry download <purl> --output <dir>`
- `codeiq registry serve --port <port>`

### `mcp`

- `codeiq mcp start --port <port> --stdio <true|false>`
- `codeiq mcp serve --port <port>`
- `codeiq mcp status`
- `codeiq mcp stop`

## 常见组合

### 第一次跑通

```bash
codeiq init
codeiq build ./sdk
codeiq diff ./sdk
```

### 发布前检查

```bash
codeiq build ./sdk
codeiq diff ./sdk --since <last-release-commit>
codeiq check ./sdk/dist/diff.json --policy ./policy/dist/bundle.ciq.tgz
```

### AI 查询

```bash
codeiq build ./sdk
codeiq mcp start
```

### 本地共享

```bash
codeiq publish ./dist/bundle.ciq.tgz
```

## 当前需要注意什么？

- `check` 当前不会自动把 `checks.policy: default-compat` 解析成一个可执行 bundle 路径；实际运行时仍建议显式传 `--policy <bundle-path>`；
- `build` 对普通 repo 和 `policy-bundle` repo 都会产出 `dist/bundle.ciq.tgz`，但 policy repo 还会额外生成 `opa-bundle.tar.gz` 和 `policy.rules.json`；
- `query` 和 `mcp` 是读本地 bundle 的入口，不负责重新 build。
