# CLI 参考

当前版本的 CodeIQ CLI 以 `codeiq.yml` 为中心：`build`、`diff`、`check`、`publish`、`registry serve`、`mcp serve` 都优先读取当前工作目录对应仓库的配置，而不是要求你在命令行里重复传路径。

---

## `codeiq init`

生成一个新的 `codeiq.yml` 模板：

```bash
codeiq init
```

默认输出到当前目录的 `./codeiq.yml`。

---

## `codeiq build`

读取当前仓库的 `codeiq.yml`，生成 `dist/` 下的 bundle 与索引产物：

```bash
codeiq build
```

典型输出：

```text
dist/
├── manifest.json
├── declarations.ndjson
├── symbols.ndjson
├── software-components.ndjson
├── sbom.spdx.json
├── metadata.json
├── checksums.txt
├── bundle.ciq.tgz
└── dependencies/              ← 仅在可解析本地依赖时出现
```

如果 `profile: policy-bundle`，还会额外生成：

```text
dist/
├── opa-bundle.tar.gz
└── policy.rules.json
```

---

## `codeiq diff`

根据当前仓库配置和 `dist/base` / `dist/target` 产物生成结构化 diff：

```bash
codeiq diff
```

输出文件：

```text
dist/diff.json
```

---

## `codeiq check`

读取 `codeiq.yml` 里的 `checks.policy` 并对 `dist/diff.json` 执行 policy 检查：

```bash
codeiq check
```

输出文件：

```text
dist/check.sarif.json
```

> `check` 依赖本机可用的 `opa eval`。

---

## `codeiq query`

一次性查询某个 bundle 中的符号或 operation：

```bash
codeiq query <query-file>
```

示例查询文件：

```json
{
  "schemaVersion": "ciq-query/v1",
  "purl": "pkg:cargo/tokio@1.43.0",
  "selector": {
    "path": "tokio::sync::Mutex::lock"
  },
  "intent": "upgrade"
}
```

`selector.kind` 当前是可选的；如果省略，CodeIQ 会在服务端做 ranked matching。

---

## `codeiq publish`

发布当前仓库 `dist/bundle.ciq.tgz` 到本地或远端 registry：

```bash
codeiq publish
```

输出文件：

```text
dist/publish.json
```

如果 `publish.includeDependencies: true`，并且 `build` 阶段生成了 dependency bundles，publish receipt 还会聚合：

- `dependencyCount`
- `dependencyReceipts`

---

## `codeiq registry`

### `registry serve`

启动本地 HTTP registry server：

```bash
codeiq registry serve --port 8787
```

默认端口来自 `codeiq.yml -> registry.port`，命令行 `--port` 会覆盖它。

本地存储目录优先级：

1. `CODEIQ_REGISTRY_STORE_DIR`
2. `codeiq.yml -> registry.storeDir`
3. `.codeiq/cache/registry`

### `registry status`

查看 registry descriptor 和健康状态：

```bash
codeiq registry status
```

---

## `codeiq mcp`

### `mcp serve`

在前台运行 MCP server：

```bash
codeiq mcp serve --port 3000
```

### `mcp status`

查看 MCP server descriptor 和健康状态：

```bash
codeiq mcp status
```

---

## 当前推荐工作流

```bash
codeiq init
codeiq build
codeiq diff
codeiq check
codeiq publish
codeiq registry serve --port 8787
```

另一个终端里，如果要给 AI / Agent 使用：

```bash
codeiq mcp serve --port 3000
```
