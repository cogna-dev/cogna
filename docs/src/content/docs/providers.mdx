# 支持范围

Cogna 目前支持以下五种仓库/规格类型。在 `cogna init` 时选择对应的 `profile`，Cogna 会按该类型提取契约信息。

---

## 支持的类型

| profile | 适用场景 |
|---------|---------|
| `go-module` | Go 语言的模块或库 |
| `rust-crate` | Rust 的 crate |
| `terraform-module` | Terraform 模块 |
| `openapi-spec` | OpenAPI 2.x / 3.x 规格文件 |
| `policy-bundle` | OPA policy 仓库（用于生成 check 使用的策略 bundle） |

---

## 各 profile 能提取和检测什么

### `go-module`

**提取内容：** 导出函数、类型、常量、接口、方法集

**能检测的变化：**

| 变化 | 说明 |
|------|------|
| 签名变化 | 函数参数或返回值变化 |
| receiver 变化 | value receiver 变成 pointer receiver |
| method set 变化 | 接口方法被删除或修改 |
| unsafe 变化 | 函数变成 `unsafe` |
| 声明删除 | 公开函数/类型被删除 |

### `rust-crate`

**提取内容：** pub 函数、类型、trait、impl、where clause

**能检测的变化：**

| 变化 | 说明 |
|------|------|
| 签名变化 | pub 函数参数或返回值变化 |
| extern ABI 变化 | `extern "C"` 函数签名变化 |
| where clause 变化 | 泛型约束变更 |
| unsafe 变化 | 函数从 safe 变成 `unsafe fn` |
| public item 删除 | pub 声明被移除 |

### `terraform-module`

**提取内容：** variable、output、module、provider、lifecycle、meta-argument

**能检测的变化：**

| 变化 | 说明 |
|------|------|
| `input-became-required` | variable 的 default 被删除 |
| `output-removed` | output 被删除 |
| `provider-ref-changed` | provider 依赖变化 |
| `lifecycle-changed` | lifecycle 配置变化 |
| `meta-argument-changed` | meta-argument 配置变化 |
| 类型变化 | variable/output 的类型声明变化 |

### `openapi-spec`

**提取内容：** paths、operations、parameters、requestBody、responses、schemas

**能检测的变化：**

| 变化 | 说明 |
|------|------|
| `parameter-became-required` | query/path/header 参数变必填 |
| `response-status-removed` | HTTP 响应状态码被删除 |
| `response-schema-narrowed` | response body 字段减少 |
| `media-type-changed` | 请求/响应的 media type 变化 |
| operation 删除 | 整个 HTTP 方法被删除 |
| path 删除 | 整个 API 路径被删除 |

### `policy-bundle`

**用途：** 把你的 `.rego` 文件构建成 OPA bundle，供 `cogna.yml -> checks.policy` 引用。

不产生 diff 或 SARIF，只生成可用于 policy 检查的 bundle 产物。

---

## 软件组件（SBOM）

所有 profile（除 `policy-bundle` 外）都会在构建时自动提取软件组件信息，生成：

- `.cogna/sbom.spdx.json`（SPDX 格式）

这些文件可以接入外部 SBOM 消费工具，也会参与 `diff` 时的 `componentChanges` 对比。

---

## 今天能做什么

- ✅ 本机 `init -> build -> diff -> check` 工作流
- ✅ 通过 SDK / Desktop / MCP 查询 API 与依赖上下文
- ✅ 生成标准 SARIF，接入 CI 和代码扫描平台
- ✅ 使用本地 cache 与 HTTP cache proxy 做远程复用
- ❌ 云端托管服务（尚未支持）
- ❌ 分布式同步和多节点（尚未支持）
- ❌ IDE 原生插件（尚未支持，见 [IDE 集成](/docs/ide)）
- ❌ Web 控制台（尚未支持，见 [Web 界面](/docs/web)）
