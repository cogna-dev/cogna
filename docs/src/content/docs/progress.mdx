## 当前产品状态

Cogna 当前以 **local-first / cache-driven** 方式工作：围绕 `cogna.yml`、本地 bundle、本地 cache、HTTP cache proxy、MCP 与 SARIF 输出形成单机闭环。A005 的设计冻结已经更新为 **package-atomic bundle cache**：`get/put` 的语义单位是完整 package bundle，而不是 manifest + 多 blob；build 在 `cache.type=http` 时采用 `local miss -> http hit -> 回填 local` 的 read-through 路径复用依赖结果。当前剩余工作主要是继续把代码、文档与辅助集成全面对齐到这份新设计真相源。

## 当前 active stage

当前 active stage 是 #strong[`A005 cache-first architecture redesign（Stage A 设计冻结）`]。当前主线工作已从上一轮 proto-first / SDK surface 收口切到下一轮存储与分发模型重构：目标是废弃 `publish` / `registry` / registry-style bundle 工作流，把远程复用统一收口到 `cache.get` / `cache.put` 两个核心动作，并明确 `get/put` 的原子单位是**一个 package 的完整 `bundle.ciq.tgz`**；build 在默认 local cache 上运行，并在 `cache.type=http` 时采用“remote read-through -> local mirror”的方式复用依赖结果，而不是重新构建。为此，本轮已完成现状盘点，并重写 `spec/cache.md` 作为新的设计真相源。

### 当前阶段目标

冻结 registry -> cache 的目标形状，明确：

1. 新的 config 应只保留 `cache`，且支持 `local` / `http` 两类 backend，并将本地落盘目录提升为 `cache.storeDir`；
2. 新的命令面应删除 `cogna publish` 与 `cogna registry ...`，改为 `cogna serve` 与 `cogna cache list/add/remove`；
3. `build` 对依赖的复用路径应改为 `local miss -> http hit -> 回填 local` 的 read-through 行为；
4. 旧的 bundle upload / receipt / versions API 应退场，由“一个 package = 一个完整 bundle object”的 cache 模型替代。
