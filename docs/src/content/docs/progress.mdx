# CodeIQ 开发进度看板

> 最后更新时间：2026-03-25（Asia/Shanghai）

此页面只保留当前有效状态，并与 [`spec/plan.typ`](https://github.com/yufeiminds/codeiq/blob/main/spec/plan.typ) 同步。当前 `R5-R9` baseline 已完成，当前 active release 为 #strong[`R10 Server Capability & Shared Framework Extraction`]：Registry 三层拆分、async HTTP runtime 与 zstd 移除已经落地，本轮剩余重点是把 MCP / Registry 的 server 化边界与验证证据完整收口到 docs/spec/dashboard。

## 已完成阶段（摘要）

- `R1-R6` 已完成并冻结为当前 repo 基线；
- `R7` 已把 Go / Rust / Terraform / OpenAPI provider evidence 收口为 repo 级证据；
- `R8` 已把 shell verifier + downgrade + local/no-auth trust baseline 固化；
- `R9` 已把 remote auth header、HTTPS gate、curl transport env 与 TLS material env contract 固化；
- 当前唯一 active release 为 `R10`，本页只对 `R10` 保留详细执行清单。

## proposal gap 快照（与 `spec/proposal.typ` 对照）

- #strong[MCP gap]：`mcp start / status / stop / serve` 已切到真实 HTTP runtime，`src/lib/mcp-sdk/*` 也已回到 transport-agnostic parser/runtime 边界；`src/cmd/mcp/main_test.mbt`、`src/e2e/review/lifecycle_test.mbt` 与全量 `moon test` 已跑通，当前剩余 gap 主要是 release 收口与下一阶段规划，而不是 runtime 正确性。
- #strong[Registry gap]：本地 / loopback HTTP server 已具备 publish / resolve / versions / bundle detail / bundle download 最小真实 API；`src/lib/registry/*`、`src/registry/*`、`src/cmd/registry/*` 的三层边界与 async HTTP transport 已落地，当前剩余 gap 主要是把 proposal/plan/docs wording 与验证证据完全对齐。
- #strong[框架拆分 gap]：Registry 边界已经固定为 `src/lib/registry` / `src/registry` / `src/cmd/registry`；当前剩余工作主要是 MCP runtime 的文档/测试收口，以及统一 dashboard 对外叙述。
- #strong[ops gap]：MCP / Registry 都已有 health contract；剩余 ops gap 主要是更系统化的 session / supervision / hosted deployment，而这些不阻塞当前 R10 最小目标。

## R10 · Server Capability & Shared Framework Extraction（Active）

**Scope**

- 把 MCP / Registry 从 “命令可用 + local/inproc/stdio workaround” 推进到 #strong[真实 server capability]；
- 把 client / server 公共框架抽到 `src/lib/*`，而不是继续混在 `src/cmd/*`；
- 保持 CLI entry 继续留在 `src/cmd/*`，同时把 Registry server/storage concrete implementation 固定在 `src/registry/*`；
- 用 proposal gap 作为本轮规划输入，统一 proposal / plan / progress 的下一阶段边界。

**Done when**

- `mcp start / status / stop / serve` 管理的已经是 #strong[真实 server runtime]，而不是 detached-stdio workaround；
- `registry` 已提供 proposal 级别的最小真实 API / runtime 证据，而不是只依赖 env stub / inproc shortcut；
- Registry 已完成 `src/lib/registry`（shared contracts/helpers + generic async client）、`src/registry`（server/storage implementation）、`src/cmd/registry`（thin CLI wiring）三层边界；
- build/query/diff/bundle reader 主链路已完全删除 zstd，统一只消费/产出 plain `*.ndjson`；
- `spec/proposal.typ`、`spec/plan.typ`、`progress.mdx` 对当前 active release 的边界一致。

**Verify**

- `moon test src/cmd/mcp/main_test.mbt --target native -v`
- `moon test src/cmd/mcp/core/main_test.mbt --target llvm -v`
- `moon test src/e2e/review/lifecycle_test.mbt --target native -v`
- `moon test src/cmd/registry/main_test.mbt --target wasm -v`
- `moon test src/cmd/publish/main_test.mbt --target wasm -v`
- `moon test src/e2e/registry_test.mbt --target wasm -v`
- `moon test src/e2e/registry_server_test.mbt --target native -v`
- `moon check`
- `pnpm build`（在 `docs/` 下）

**Out of scope**

- shared non-shell verifier；
- trust root / key rotation / KMS；
- multi-node hosted registry productization、HA、enterprise observability；
- 把 `build / diff / check / publish` 迁移进 MCP。
