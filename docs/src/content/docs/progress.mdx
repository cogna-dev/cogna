## R7 · Provider Matrix Coverage（Completed）

- 已把 Go / Rust / Terraform / OpenAPI provider 能力矩阵冻结为 repo 证据；

- 详细执行清单不再保留在本页；当前没有 active release，后续只保留 backlog / 下一阶段摘要。

## R8 · Registry Trust & Verifier（Completed）

- 已统一 `build → publish → registry` 的 signature metadata 语义，并把 `verified / untrusted / unsigned` 的 downgrade 边界冻结为 repo baseline；
- 已正式确认 shell OpenSSL CLI 继续作为当前 verifier baseline，继续依赖 `CODEIQ_SIGNATURE_PUBLIC_KEY_FILE`；
- 已确认 local registry、`inproc://registry` 与 loopback/开发场景允许免认证 / 降级运行，非本地 remote registry 继续执行 stricter trust enforcement；
- shared non-shell verifier、trust root、KMS / key rotation、hosted deployment 继续保留在 backlog，不阻塞当前阶段。

## R9 · Deployment & Security Hardening（Completed）

- 已为 remote registry 固化 `CODEIQ_REGISTRY_HEADER_*` header/token 注入边界；
- 已为 remote endpoint / `downloadUrl` 固化 transport security baseline：默认要求 `https://`，只有显式设置 `CODEIQ_REGISTRY_ALLOW_INSECURE_HTTP=true` 时才允许非本地 `http://`；
- 已为 curl transport 固化可配置 env：`CODEIQ_REGISTRY_CURL_MAX_TIME`、`CODEIQ_REGISTRY_CURL_CONNECT_TIMEOUT`、`CODEIQ_REGISTRY_CURL_RETRY`、`CODEIQ_REGISTRY_CURL_INSECURE_TLS`；
- 已为 optional TLS 材料固化 env：`CODEIQ_REGISTRY_TLS_CA_FILE`、`CODEIQ_REGISTRY_TLS_CLIENT_CERT_FILE`、`CODEIQ_REGISTRY_TLS_CLIENT_KEY_FILE`；
- 详细执行清单不再保留在本页，后续只把 backlog / 下一阶段保留为摘要方向。

## 当前 trust baseline 结论

- 当前 repo 已经不是“尚未开始 trust/verifier / deployment baseline”状态，而是 #strong[R8 / R9 baseline 已完成并冻结]：
  - `build` 已携带 signature provenance；
  - `registry` / `publish` 已有 OpenSSL shell verify path；
  - schema / tests 已覆盖 verified / untrusted / downgrade / remote propagation 语义；
- `R8` 当前正式决定：继续保留 shell verifier，并以 OpenSSL CLI + downgrade 作为当前 baseline，而不是在本轮切换到 shared non-shell verifier；
- `local registry`、`inproc://registry` 与 loopback 开发场景允许免认证运行；非本地 remote registry 仍维持 stricter trust enforcement；
- `R8` 的重点不是从零开始发明 trust model，而是把现有 baseline 收敛成清晰、稳定、可决策的实现边界；
- 当前外部参考可作为术语基线：`OCI Distribution` 的 digest verification、`cosign` 的 detached signature / bundle、`TUF/Rekor` 的 trust metadata / transparency；但这些并不自动意味着本轮就实现完整 shared verifier；
