## R8 · Registry Trust & Verifier（Active）

**Scope**

- 在 provider matrix baseline 与 native workaround 都已稳定后，正式收口 `signature path`、`registry trust` 与 `verifier feasibility`；
- 对齐 `build → publish → registry` 三条链路中的 signature metadata 语义，而不是继续停留在“只有 schema 与测试”的规划状态；
- 明确当前 shell OpenSSL CLI 校验路径是否继续保留为过渡实现，以及 shared non-shell verifier 是否进入下一轮实施。
- 用 `OCI digest verification`、detached signature / bundle、local / remote verification 等外部成熟术语约束 R8 表述，避免把 backlog trust 议题重新混入当前 release。

**当前 repo 基线**

- `src/cmd/build/main.mbt`
  - 已通过 `runtime_signature_from_env()` 把 `CODEIQ_SIGNATURE_*` 注入 manifest provenance；
- `src/cmd/registry/main.mbt`
  - 已实现 `verify_signature_for_bundle_file(...)`，当前通过 shell 调用 OpenSSL 做 base64 decode + `openssl dgst -sha256 -verify`；
- `src/cmd/publish/main.mbt`
  - 已实现 `verify_signature_with_openssl(...)`，并在 receipt / remote download signature 上执行 trust downgrade / reject 语义；
- `src/schema/contracts.mbt`
  - 已把 upload / download / publish receipt 的 `signature` shape 固化为 repo contract；
- `src/cmd/registry/main_test.mbt` 与 `src/cmd/publish/main_test.mbt`
  - 已覆盖 verified / untrusted / missing-value downgrade / remote propagation 等 trust baseline 行为。

**Done when**

- `build` / `publish` / `registry` 对 signature status、algorithm、keyId、value 的行为边界有统一表述；
- 当前 OpenSSL shell verifier 的保留范围、局限与替换条件被明确写入 docs / plan / progress；
- shared verifier feasibility 被收敛为明确结论：
  - 要么进入下一轮实现；
  - 要么被正式降级到 backlog，并写清原因；
- trust root / key rotation / KMS / hosted registry 不再混入当前 release，而是保持清晰 backlog 边界。

**Verify**

- `moon test src/cmd/registry/main_test.mbt --target wasm -v`
- `moon test src/cmd/publish/main_test.mbt --target wasm -v`
- `moon test src/schema/contracts_test.mbt --target native -v`
- `pnpm build`（在 `docs/` 下）

**Out of scope**

- shared non-shell verifier 的完整实现；
- trust root / key rotation / KMS；
- hosted registry productization、auth、deployment、enterprise security。
