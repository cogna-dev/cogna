## 当前产品范围（scope reset）

- #strong[单机优先]：只面向单台机器上的 CLI / Local Registry / MCP 工作流；
- #strong[共享本地存储]：本地 bundle cache 目录是唯一 authoritative storage；
- #strong[不支持 trust verifier]：不再把 trust verifier、trust root、KMS 写入当前产品范围；
- #strong[不支持 distributed / hosted]：不再把 remote registry、hosted deployment、多节点 HA 写入当前产品范围；
- #strong[新增 `src/lib/sbom/*`]：从源码与包管理器文件提取软件成分，导出 CycloneDX / SPDX，并把软件成分信息纳入 bundle 构建；
- #strong[继续推进语义和 policy]：在本地边界内继续推进 provider semantic depth、CIQ schema richness 与 OPA diff。

## 已完成阶段（摘要）

- `R1-R6` 已完成并冻结为当前 repo 基线；
- `R7` 已把 Go / Rust / Terraform / OpenAPI provider evidence 收口为 repo 级证据；
- `R8-R9` 代表历史 trust / remote hardening baseline，但不再继续作为当前产品方向；
- `R10` 已完成 MCP / Registry 的本地 server capability 与共享框架抽取；
- `R12` 已完成 provider semantic depth、consumer readiness 与 `R13` handoff；
- `R13` 已完成 semantic diff、component diff、OPA input shaping 与 SARIF enrichment 的本地闭环；
- `spec/plan.typ` 已新增 proposal → implementation reconciliation，对 proposal 中的历史样例、aspirational wording 与当前 repo reality 做了显式对账；
- 当前尚未开启新的 active release，因此本页不再保留详细执行清单。

## R12 → R13 handoff 结论

- #strong[stable identity 不变]：`id / kind / path / signature / location` 仍是当前 declaration matching / traceability contract。
- #strong[consumer payload additive]：`shape`、`language_specific`、`name`、`canonical_name`、`language`、`baselineRole`、`source_refs`、`relations`、`visibility`、`docs` 现在都可以由 bundle/query/diff 稳定 transport。
- #strong[query handoff 已明确]：`ciq-result/v1.matches[*]` 保持原 selector 语义不变，但允许 richer declaration payload 直接透传给下游 consumer。
- #strong[diff handoff 已明确]：`ciq-diff/v1.changes[*]` 继续使用 stable keys/matching，同时把 richer facts 以 `beforeRecord` / `afterRecord` 形式作为 additive evidence 暴露给后续 semantic diff。
- #strong[semantic interpretation defer to R13]：provider semantic diff、SBOM diff、OPA input shaping 与 breaking/policy judgement 仍属于 `R13`，不在 `R12` 内提前实现。

当前没有新的 active release；`R13` 已完成并作为最新已验证基线收口。

## R13 · OPA Diff Enrichment（Completed summary）

**Goal**

- 消费 `R12` 已稳定的 richer declaration / software-component facts，构建更可靠的 semantic diff 与 policy input；
- 让 `CIQ Diff` 和 OPA input 从浅层 changed/add/remove marker 升级到 declaration diff + provider semantic diff + SBOM diff 的统一语义面；
- 保持本地 bundle store / Local Registry 范围，不重新引入 verifier、remote 或 hosted 能力。

**R12 → R13 handoff matrix**

| Surface | R12 guarantees now | R13 interprets later |
| --- | --- | --- |
| Stable identity | `id` / `kind` / `path` / `signature` / `location` 保持为 matching / traceability contract | 是否构成 breaking / warning / allow / deny judgement |
| Bundle declarations | richer additive facts 已稳定落盘并被 bundle reader 接受 | 如何把这些 facts 聚合为 provider semantic diff / SBOM diff |
| `ciq-result/v1.matches[*]` | 透传 `path`、`name`、`canonical_name`、`language`、`baselineRole`、`source_refs`、`relations`、`visibility`、`docs`、`shape`、`language_specific` | 如何基于 richer payload 做 query-time semantic views / policy-oriented projections |
| `ciq-diff/v1.changes[*]` | `changed` 带 `beforeRecord` + `afterRecord`；`added` / `deprecated` 带 `afterRecord`；`removed` 带 `beforeRecord` | 如何把 richer before/after evidence 转成 semantic categories、policy input 与 SARIF/OPA 结果 |

**Completed outcomes**

- `src/cmd/diff/main.mbt` 现在对 declaration changes 输出稳定的 `semanticCategory` 与 `semanticDiff`，并对 component changes 输出 `componentDiff`；
- component diff 现在按稳定 component identity 做匹配，不再把纯版本升级误判为 remove + add；
- `src/cmd/check/main.mbt` 现在把 diff 整形成 OPA-friendly input（`changes` + `componentChanges`），并在 SARIF 中使用 `semanticCategory` 与 declaration record location enrichment；
- `src/schema/contracts.mbt` 已显式接受 `componentDiff`，`src/schema/contracts_test.mbt` 已覆盖对应 contract；
- `src/cmd/check/main_test.mbt` 已验证真实 diff record 的 `location.uri` / `startLine` 会进入 SARIF。
