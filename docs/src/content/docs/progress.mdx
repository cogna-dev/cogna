## 当前产品状态

CodeIQ 当前仍以 **local-only / config-driven** 方式工作：围绕 `codeiq.yml`、本地 bundle、本地 registry store、MCP 与 SARIF 输出形成单机闭环。A003 的 benchmark / parity 与 boundary cleanup 已完成，native-only async 收口也已经落地；A004 的 JS SDK/CLI 边界第一批实现也已落地：`src/sdk/contracts/`、`src/sdk/index/`、`src/sdk/query/` 已创建，`cmd/query` / `cmd/build` / `cmd/mcp/core` 已完成对 SDK core 的薄封装或委托，`integrations/sdk/node/` 也已收口为带稳定 wrapper API 的 `@codeiq/sdk`，同时 repo 根目录已补 `pnpm-workspace.yaml`，公开 TS 类型已从大量 `unknown` 收紧为 CIQ/local-index 相关结构。在此基础上，当前主线工作已切到：#strong[A004 proto-only 丰富 contract review]。本轮继续在五文件布局内重构 proto：共享文件保持为 `bundle.proto`，而 `query.proto` 已开始从旧的 selector/result 模型转向“Bundle 虚拟二维表 + SQL”模型，用 `BuiltinTable`、`TableSchema`、`ColumnSchema`、`QueryValue`、`QueryResultSet` 和 `QueryCatalog` 来定义可查询的关系视图，减少后续继续堆叠 `List/Get` 接口的需要。
