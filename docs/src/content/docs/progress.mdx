## 已完成阶段（摘要）

- `R1-R6` 已完成并冻结为当前 repo 基线；
- `R5` 已完成，主业务流程 `init → build → diff → check → query → MCP review` 已形成稳定闭环；
- `R6` 已完成，单机/免认证 registry + publish baseline 已关闭；
- 当前看板记录最新完成阶段与当前 active release：`R7`、`R8` 已完成，`R9` 已成为当前执行阶段。

## R7 · Provider Matrix Coverage（Completed）

- 已把 Go / Rust / Terraform / OpenAPI provider 能力矩阵冻结为 repo 证据；
- `examples/*`、extractor、e2e tests 与 `providers / progress / plan` 文档已完成收口；
- 详细执行清单不再保留在本页，当前 dashboard 只展开 active release `R8`。

## R8 · Registry Trust & Verifier（Completed）

- 已统一 `build → publish → registry` 的 signature metadata 语义，并把 `verified / untrusted / unsigned` 的 downgrade 边界冻结为 repo baseline；
- 已正式确认 shell OpenSSL CLI 继续作为当前 verifier baseline，继续依赖 `CODEIQ_SIGNATURE_PUBLIC_KEY_FILE`；
- 已确认 local registry、`inproc://registry` 与 loopback/开发场景允许免认证 / 降级运行，非本地 remote registry 继续执行 stricter trust enforcement；
- shared non-shell verifier、trust root、KMS / key rotation、hosted deployment 继续保留在 backlog，不阻塞当前阶段。

## R9 · Deployment & Security Hardening（Active）

**Scope**

- 在 `R8` trust baseline 固定后，继续推进 remote registry auth boundary、hosted deployment、transport / storage security 与 ops hardening；
- 让当前本地 / 单机语义逐步扩展到可部署、可鉴权、可运维的更完整 runtime boundary；
- 保持已完成的 shell verifier + downgrade baseline 不回退，同时避免把 shared verifier / KMS backlog 与当前部署问题混写。

**Done when**

- remote registry 的认证 / 鉴权边界、header / token 注入方式与 CLI 行为被明确固定；
- deployment / transport / storage security 的最小可验证基线在 docs 与实现中一致；
- ops / hosted registry productization 的下一轮边界被明确，不再与 R8 trust baseline 混淆。

**Verify**

- `moon test src/cmd/registry/main_test.mbt --target wasm -v`
- `moon test src/cmd/publish/main_test.mbt --target wasm -v`
- `moon test src/e2e/registry_test.mbt --target wasm -v`
- `pnpm build`（在 `docs/` 下）

**Out of scope**

## 当前 trust baseline 结论

- 当前 repo 已经不是“尚未开始 trust/verifier”状态，而是 #strong[baseline 已完成并冻结]：
  - `build` 已携带 signature provenance；
  - `registry` / `publish` 已有 OpenSSL shell verify path；
  - schema / tests 已覆盖 verified / untrusted / downgrade / remote propagation 语义；
- `R8` 当前正式决定：继续保留 shell verifier，并以 OpenSSL CLI + downgrade 作为当前 baseline，而不是在本轮切换到 shared non-shell verifier；

- `R8` 的重点不是从零开始发明 trust model，而是把现有 baseline 收敛成清晰、稳定、可决策的实现边界；
