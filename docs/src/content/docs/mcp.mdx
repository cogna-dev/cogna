# MCP 在 CodeIQ 中扮演什么角色？

CodeIQ 的 MCP Agent 是一个本地常驻内存查询代理。它的作用不是构建 bundle，也不是执行差异比较或策略检查，而是在 bundle 已经准备好的前提下，为 AI 客户端提供高频查询能力。

在当前实现中，它更准确的角色仍是：#strong[review consumption surface]。也就是先由 CLI 生成 bundle、diff、SARIF、query result，再由 MCP 为审查者或 Agent 暴露稳定的查询入口；这条查询入口已经从 stdio / detached-stdio baseline 迁移到真实 server runtime，并与 CLI / Local Registry 共享同一套本地 bundle cache。

在这个模型里：

- `build` 负责生成 `CIQ Bundle` 或 `CIQ OPA Bundle`
- `mcp start` 负责把本地 bundle 装载到内存并暴露查询能力
- `publish` 负责把稳定 bundle 写入共享本地 Local Registry，供同机 CLI / MCP 复用

## 一个典型的本地流程

```bash
codeiq init
codeiq build ./sdk
codeiq mcp start
```

这个流程适合以下情况：

- 你在本机验证一个仓库
- 你希望 IDE、终端 Agent 或本地工具直接查询上下文
- 你暂时不需要团队级共享

## MCP 为什么重要？

因为它把“产物构建”和“上下文消费”拆成了两个阶段：

1. **先构建**：把仓库整理成标准化 bundle。
2. **再查询**：把这些 bundle 以低延迟方式提供给 AI 客户端。

这样做的好处是：

- 同一个 bundle 可以被同机上的多个客户端复用。
- AI 不必每次都重新解析仓库。
- 查询入口更可控，也更容易做缓存和版本绑定。

当当前工作目录没有现成 `dist/` bundle 时，MCP 也可以按 PURL 从共享本地 registry store 物化 bundle 到 `.codeiq/cache/mcp/<purl-key>/dist`，再继续提供查询。

## MCP 当前提供哪些工具？

CodeIQ 当前保留两个 MCP 工具：

### `codeiq.query.outlines`

根据 PURL 返回指定包的公开接口大纲索引。适合让 AI 先了解“这个包大概有哪些公开内容”。

典型用途：

- 先列出候选模块、类型、函数、路径
- 再决定下一步应查询哪个具体 symbol

### `codeiq.query.symbol`

在已有 path 的前提下，对具体声明做精确查询，返回：

- 声明 ID
- 签名
- 注释
- 位置
- Terraform / OpenAPI 契约摘要

## 推荐查询顺序：先 outlines，后 symbol

推荐 AI Agent 使用如下顺序：

1. 调用 `codeiq.query.outlines` 获取公开接口大纲。
2. 从返回的大纲里选择目标 path。
3. 再调用 `codeiq.query.symbol` 获取精确声明与契约细节。

这样比直接盲查单个 symbol 更稳定，也更适合 AI 在大包中逐步缩小范围。

## 什么时候需要 `status` 与 `stop`？

```bash
codeiq mcp status
codeiq mcp stop
```

- `status`：检查当前服务是否可用，适合排查连接问题或自动化探活。
- `stop`：在本地开发或脚本收尾阶段关闭服务。

如果需要以前台方式直接运行 MCP server，可使用：

```bash
codeiq mcp serve
```

该模式当前会启动真实的本地 HTTP JSON-RPC runtime，提供：

- `GET /health`
- `POST /rpc`

`codeiq mcp start` / `status` / `stop` 管理的也是同一套 runtime state 与 health contract；兼容参数 `--stdio` 仍被接受，但不再切回旧的 detached stdio baseline。

## 本地 MCP 与 CLI 的边界

| 能力 | 应该通过什么入口 |
| --- | --- |
| 构建 bundle | CLI |
| 生成 diff | CLI |
| 执行 check（OPA CLI policy decision）并输出 SARIF | CLI |
| 发布到 Registry | CLI |
| 查询 outlines / symbol | MCP 或一次性 CLI query |

简而言之：MCP 只负责查询，不负责构建和治理。
