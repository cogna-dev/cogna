# AI 查询

这一页属于 #strong[How-to]：当你要让 AI / Agent 查询接口时，直接照这里做。

## 两种方式

| 场景 | 入口 |
| --- | --- |
| 一次性查询 | `codeiq query` |
| 多轮交互式查询 | `codeiq mcp start` |

它们的边界很明确：

- `query` 适合脚本、单次调试、CI 里的离线读取；
- `mcp start` / `mcp serve` 适合本地 AI / Agent 多轮查询；
- `build`、`diff`、`check`、`publish` 仍然应该通过 CLI 完成，而不是通过 MCP 触发。

## 推荐顺序

1. `codeiq.query.outlines`
2. `codeiq.query.symbol`

这样更容易先缩小范围，再看细节。

## MCP 常用命令

```bash
codeiq mcp start
codeiq mcp status
codeiq mcp stop
codeiq mcp serve
```

当前 MCP runtime 会优先读取当前工作目录下的 `dist/manifest.json`；如果你传入的 PURL 不在当前目录，它会尝试从本地 registry / cache materialize 对应 bundle，而不是走远端 hosted query。

## MCP tools

- `codeiq.query.outlines`
- `codeiq.query.symbol`
- `codeiq.runtime.context`

推荐的调用顺序是：

1. 先用 `codeiq.query.outlines` 浏览某个 PURL 的公开接口大纲；
2. 再用 `codeiq.query.symbol` 精确读取某个 path / selector 的签名、位置、docs、shape、relations 和 `language_specific`；
3. 需要调试 runtime 状态时，再看 `codeiq.runtime.context`。

## 一个最小的调用心智

- 先用 `outlines` 找候选路径
- 再用 `symbol` 查具体签名、位置和说明

这套心智同样适用于 Terraform / OpenAPI：先缩小 contract 范围，再精确读取 declaration facts。
