# AI 查询（MCP）

Cogna 通过本地 MCP server 把 bundle 内容暴露给 AI 工具。当前命令面已经收口为：

- `cogna mcp serve`
- `cogna mcp status`

也就是说，MCP 现在是一个**前台服务进程**；是否由外部 supervisor、容器或 IDE 负责后台常驻，由你自己决定。

## 两种查询方式

| 使用场景 | 命令 |
|----------|------|
| AI / Agent 查询（单轮或多轮） | `cogna mcp serve --port 3000` |

---

## 快速开始

```bash
# 1. 先构建 bundle
cogna build

# 2. 在前台启动 MCP server
cogna mcp serve --port 3000

# 3. 另一个终端里查看状态
cogna mcp status
```

---

## MCP 提供的工具

### `cogna.query.outlines`

按 PURL 返回公开路径大纲，适合先缩小范围。

### `cogna.query.symbol`

按路径读取具体声明。`kind` 当前是可选的；如果不传，服务端会做 ranked matching。

### `cogna.runtime.context`

返回当前 runtime 看到的 bundle / cache 上下文，适合调试和确认加载状态。

---

## 与 Cache 的关系

MCP 自己不负责构建 bundle。常见做法是：

1. `cogna build`
2. `cogna serve --port 8787`
3. `cogna mcp serve --port 3000`

这样 CLI、cache proxy、MCP 三者都围绕同一份本地 bundle / store 工作。
